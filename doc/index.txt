#] HOW STUFF IS DOCUMENTED
Generally, in files, it will be documented with the following template:
CODE---
DESCRIPTION ---
    Description of the class

PARAMS ---
    param\_name [param_type]
    - Description of this param, eg what it does
    .
    .
    .
    >> If a class shouldn't be initialized by hand, a message will be there instead

FUNCTIONS ---
    function(...)
    DESCRIPTION ---
        What this function does
    PARAMS ---
        param\_name [param_type]
        - Description of this param
    RETURNS ---
        return_type - Description of what it is maybe
---
Also, to get a specific file, add `?{file\_name}` to the url where `{file\_name}` is what is shown in the `[FILE]` tab
Additionally, to get a specific section, add `#{sect\_name}` where `{sect\_name}` is what is shown in the `[JUMP]` tab
Lastly, to find a specific word, add `&{word}` where `{word}` is any word or phrase you like
